Mobile Ask Citation Review

Summary. 用户反馈显示，移动端 Ask 的主要摩擦来自答案可信度确认，而不是单纯的输入效率。用户在得到回答后，需要快速查看来源文件、页码和原文片段。

调研中，多数用户会在答案完成后立刻寻找引用入口。如果引用入口被折叠或需要返回上一层页面，用户会中断当前问题链路，导致后续追问减少。

2 建议将引用片段作为答案下方的第一层内容，并在预览页保留问题、答案、citationIndex 与返回状态。预览不仅展示原文，还要解释为什么该片段被引用。

对于 OCR 图片和 PDF，预览需要同时支持 bbox 高亮、页码切换、缩放和全屏阅读。高亮区域应该在视觉上足够明确，但不要遮挡原文可读性。

Answer Citation Preview

最终目标是让证据链保持连续：从 Search 或 Ask 打开预览后，用户仍然知道自己为什么来到这里、正在看第几条引用，以及如何回到原问题。

WHY THIS CITATION #2

该片段直接说明引用回看需要保留问题、答案和 citationIndex。 它匹配当前搜索问题“移动端 Ask 页面如何优化引用回看”，并解释预览页的核心价值。
